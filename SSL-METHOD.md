# Small Systems Lab Method

Small Systems Lab is not only a collection of projects. It is a method for building public-interest systems from care, evidence, access, and accountable rules.

## Core Method

### 1. Operations of Care
Every project must define how the system works in practice:
- who can use it;
- what it collects;
- what it refuses to collect;
- how consent is recorded;
- how risk is escalated;
- how information is preserved, withheld, corrected, or shared;
- who has authority to override the machine.

### 2. Rule-Based Intelligence
SSL treats intelligence as governance, not magic. The system should expose its rules:
- Human observed;
- Machine suggested;
- Context missing;
- Consent unclear;
- Protected uncertainty;
- Unsafe to publish;
- Ready for review;
- Withheld.

The machine may assist, classify, translate, summarize, or detect patterns, but it does not replace human judgment.

### 3. Ancient Geometry as Interface Logic
Ancient geometry is used as orientation and structure, not decoration.
- Circle: consent, return, community review, recurrence.
- Triangle: witness, source, context.
- Grid: taxonomy, archive, navigation, civic order.
- Spiral: memory, time, learning, revision.
- Golden ratio: proportion, calm, readable interface hierarchy.
- Threshold/gate: publish, withhold, verify, escalate.

### 4. Accessibility as System Architecture
Accessibility is not an afterthought. It is the operating system:
- keyboard-first navigation;
- screen-reader-readable structure;
- alt text and captions;
- high contrast;
- plain-language summaries;
- transcripts;
- multilingual pathways;
- low-bandwidth and offline-first assumptions.

### 5. Public Knowledge and Returnability
A system is successful only if information can return to the people it concerns in a useful, respectful, understandable form.
