# Blind Editor Workflow

A blind editor must be able to:

1. review an observation
2. hear or read transcript
3. inspect consent state
4. inspect risk state
5. inspect source state
6. edit alt text
7. approve or hold publication

## Relationship to Web Engine Screens

This workflow maps onto the review screens in `web-engine/screens.md` (observation detail, source review, consent review, risk review, accessibility review, human decision, publication status). None of those screens may be keyboard- or screen-reader-inaccessible.
