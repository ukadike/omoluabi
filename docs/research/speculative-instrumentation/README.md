# Speculative Instrumentation Research

This directory documents a repeatable research method used on Omoluabi: building speculative future scenarios in UMADA with enough historical and material detail that they expose weaknesses in present-day observation instruments, then translating those weaknesses into Omoluabi technical requirements.

## Files

1. [`00-method-overview.md`](00-method-overview.md) — what the method is and why it's used
2. [`01-photographic-witness-case-study.md`](01-photographic-witness-case-study.md) — first completed case study
3. [`02-cartographic-witness-case-study.md`](02-cartographic-witness-case-study.md) — second completed case study
4. [`03-speculative-to-technical-traceability-matrix.md`](03-speculative-to-technical-traceability-matrix.md) — scenario → MVP change table
5. [`04-design-decisions-log.md`](04-design-decisions-log.md) — individual decision records
6. [`05-future-instrumentation-questions.md`](05-future-instrumentation-questions.md) — research backlog, not MVP commitments

## Status

All documents in this directory are research documentation, not the schemas or plans themselves. Where this directory says a technical requirement was "discovered," the corresponding schema, card, or plan document is the artifact of record — this directory explains why it exists.
