# Omoluabi: One-Page Brief

**What it is:** An open, rules-based editorial intelligence system for trustworthy observation, consent, provenance, accessibility, civic memory, and public knowledge.

**Why now:** AI systems can generate, summarize, classify, and distribute information faster than governance, consent, accessibility, and provenance can keep up. Omoluabi starts from governance first, not as an afterthought.

**What it produces:**

- a schema and rule library (`schemas/`, `cards/`, `governance/`)
- a physical observation device MVP plan (`device/`)
- a web engine MVP plan (`web-engine/`)
- a public API roadmap (`api/`)
- accessibility-first workflows (`accessibility/`)
- a speculative stress-test sandbox, UMADA (`umada-sandbox/`)
- a bridge to scientific/environmental observation, Earth Sensors Lab (`earth-sensors-lab-bridge/`)

**Who benefits:** journalists, community archivists, researchers, educators, accessibility practitioners, environmental observers, civic institutions, museums, public-interest technologists.

**Status:** research architecture with working schema and documentation MVPs; device and web engine are still plans, not built prototypes.
