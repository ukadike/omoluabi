# Reviewer-Facing Summary

This repository is written to be legible to: SSRC reviewers; Mozilla, Ford, Knight, Sloan, and Mellon-style reviewers; Processing Foundation collaborators; journalists; open-source maintainers; accessibility researchers; civic technologists; AI safety researchers; HCI researchers; museums; archivists; universities; public-interest technology labs; hardware collaborators; and future API users.

## What to Look For

- **Governance before AI:** see `governance/principles.md` and `governance/rule-library.md`.
- **Schemas before apps:** see `schemas/` and `cards/` — every interface idea is backed by a JSON Schema first.
- **Provenance and consent:** see `schemas/source.schema.json`, `schemas/consent.schema.json`, and the open reconciliation question noted in `governance/consent-model.md`.
- **Accessibility as infrastructure, not polish:** see `accessibility/baseline.md`.
- **Honesty about what is unscoped:** federation, local-first sync, and licensing are explicitly marked draft/pending rather than fabricated (see `architecture/sync-and-federation.md`, `web-engine/local-first-plan.md`, `LICENSE`).

## Recognition Strategy

Recognition should come from transparent research, reproducible schemas, accessible documentation, clear architecture, working prototypes, strong public-interest framing, and disciplined governance — not from polish alone.

## What This Repository Is Not

It is not a finished product. The MVP is a working governance loop (`docs/mvp-plan.md`), not a finished device or finished web engine.
