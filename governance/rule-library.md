# Rule Library

## Rule ID Format

```text
SSL-001
OM-001
ESL-001
ECHO-001
UMADA-001
```

SSL is used for rules shared across Small Systems Lab projects. OM is reserved for rules specific to Omoluabi. No OM-prefixed rules have been written yet — all current rules are shared SSL rules.

## Starter Rule Library

### SSL-001 — Observation First
Every record begins as an observation, not a conclusion.

### SSL-002 — Provenance Required
Every observation must identify source, origin, device, collector, or import method.

### SSL-003 — Consent State Required
Every observation must declare consent state before sharing.

### SSL-004 — Accessibility Required
No public object is complete without accessibility metadata.

### SSL-005 — AI Is Assistive
AI may suggest, summarize, transcribe, translate, classify, or draft. AI may not publish, erase, override, or govern.

### SSL-006 — Human Publication
A human must approve public release.

### SSL-007 — Correction Visibility
Corrections must preserve what changed, when, why, and by whom.

### SSL-008 — Risk Before Reach
Distribution must be constrained by risk review.

### SSL-009 — Local First
Field observations should be creatable without internet access.

### SSL-010 — API Readiness
Stable schemas should be designed as future public API objects.
